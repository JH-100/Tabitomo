# manage-issues

GitHub Issues를 관리합니다. gh CLI를 사용합니다.

## 사용법

- `/manage-issues create <제목>` - 새 이슈 생성
- `/manage-issues update <번호> <상태메시지>` - 이슈 업데이트
- `/manage-issues close <번호>` - 인수조건 확인 후 이슈 닫기

## 이슈 생성 규칙

1. 제목은 명확하고 간결하게
2. 본문에 반드시 포함:
   - **배경:** 왜 이 작업이 필요한지
   - **작업 개요:** 무엇을 할 것인지
   - **인수조건:** 완료 기준 (체크리스트)
   - **의존 관계:** 선행 작업 이슈 번호
3. 적절한 라벨 부여

## 이슈 닫기 규칙

1. 인수조건을 하나씩 확인
2. 모든 인수조건 충족 시에만 닫기
3. 닫기 코멘트에 인수조건 확인 결과 포함
