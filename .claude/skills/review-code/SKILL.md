# review-code

코드 리뷰를 수행하고 개선점을 제안합니다.

## 사용법

`/review-code [파일경로 또는 영역]`

## 리뷰 체크리스트

1. **보안:** 입력 검증, 인증/인가, 민감 데이터 노출
2. **성능:** 불필요한 리렌더링, 메모리 누수, N+1
3. **코드 품질:** DRY, SRP, 네이밍, 복잡도
4. **에러 처리:** 예외 처리, 사용자 피드백
5. **접근성:** 스크린 리더, 색상 대비
6. **타입 안전:** any 사용, 타입 가드

## 출력 형식

- CRITICAL: 즉시 수정 필요
- HIGH: 가능한 빨리 수정
- MEDIUM: 개선 권장
- LOW: 선택적 개선
